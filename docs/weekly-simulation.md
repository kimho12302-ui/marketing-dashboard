# 2년차 마케터 주간 대시보드 사용 시뮬레이션 (12라운드)

> 시뮬레이션 주체: 2년차 콘텐츠 마케터 (PPMI 소속)
> 시뮬레이션 단위: 1주일 = 1라운드
> 목표: 매주 대시보드로 데이터 입력 → 분석 → 리포팅하면서 문제점 찾기

---

## 라운드 1: 첫 주 — "어디서부터 시작하지?"

### 시나리오
월요일 출근. 지난주 데이터를 대시보드에 넣어야 한다.

### 수행 과정
1. Settings → 일일 입력 탭 접속
2. 날짜를 금요일로 변경 → GFA 광고비 입력 → 저장
3. 날짜를 목요일로 변경 → GFA 광고비 입력 → 저장
4. 쿠팡 광고 파일 업로드 → 성공
5. 판매 엑셀 업로드 → 성공
6. 싱크 버튼 클릭
7. Overview 확인

### 발견된 문제점
1. **주말 2일치를 한번에 못 넣음** — 수기 입력은 날짜 하나씩 바꿔야 함 (3일치면 3번 반복)
2. **진행 상태가 날짜별이 아님** — 체크리스트가 "오늘" 기준이라, 금/토/일 3일치 입력 상황을 반영 못함
3. **어떤 날짜 데이터가 비어있는지 모름** — 상태 패널은 "최신 날짜"만 보여줘서, 중간에 빠진 날 확인 불가
4. **입력 순서가 강제적** — "해당 없으면 체크만 하고 넘어가세요"라고 했지만, 섹션 7개를 다 봐야 함

### 개선 방안
- [ ] 수기 입력에 **날짜 범위 선택** 옵션 (같은 값 여러 날 일괄 입력)
- [ ] 데이터 상태 패널에 **최근 7일 캘린더 뷰** — 어떤 날이 비어있는지 한눈에
- [ ] 섹션 접기/펼치기 기본값: 데이터 없는 섹션만 열기

---

## 라운드 2: 둘째 주 — "숫자는 맞는 건가?"

### 시나리오
데이터 입력에 익숙해짐. Overview에서 지난주 성과를 본다.

### 수행 과정
1. Overview 접속 → KPI 카드 확인 (매출, 광고비, ROAS)
2. "이 숫자 맞나?" → 시트 열어서 대조
3. Sales 탭 확인 → 채널별 매출 차트
4. Ads 탭 → 채널별 광고비 비교
5. 주간 리포트 작성을 위해 PDF 내보내기

### 발견된 문제점
5. **시트와 대시보드 숫자가 다름** — 시트에는 있는데 DB에 없거나, 역동기화 안 돼서 시트가 비어있음
6. **KPI 카드에 기간 비교가 없음** — "지난주 대비 ↑12%" 같은 비교가 없어서 좋은 건지 나쁜 건지 모름
7. **PDF에 날짜가 안 나옴** — 어느 기간 리포트인지 불명확
8. **ROAS를 브랜드별로 못 봄** — Overview ROAS는 전체만, 너티/아이언펫 각각의 ROAS 확인하려면 필터 변경 필요

### 개선 방안
- [ ] KPI 카드에 **전주 대비 % 변화** 표시
- [ ] PDF 리포트에 **기간 명시** + 전주 비교 섹션
- [ ] Overview에 **브랜드별 ROAS 요약 테이블** 추가
- [ ] 시트↔DB 크로스 체크 기능 (or 역동기화 완성)

## 라운드 3: 셋째 주 — "광고 성과가 떨어졌는데 왜?"

### 시나리오
ROAS가 떨어진 걸 발견. 원인 파악이 필요하다.

### 수행 과정
1. Ads 탭 → ROAS 트렌드 확인 → 이번 주 하락
2. 채널별로 보려고 필터 변경
3. Creatives 확인 → 어떤 소재가 안 좋았나
4. 소재별 ROAS 비교
5. 결론 도출 → 어떤 액션을 해야 하나

### 발견된 문제점
9. **ROAS 하락 원인을 자동으로 안 알려줌** — 직접 채널→소재→기간 조합을 탐색해야 함
10. **소재 성과가 기간별 추이로 안 보임** — 소재별 일별 트렌드가 인라인이라 전체 비교 어려움
11. **채널 간 ROAS 비교가 한 화면에 안 나옴** — Ads 탭에서 필터 바꿔가며 봐야 함
12. **"다음에 뭘 해야 하나" 가이드가 없음** — ROAS 떨어지면 어떻게 하라는 액션 제안이 없음

### 개선 방안
- [ ] **자동 원인 분석 강화** — "ROAS 하락 원인: 너티 메타 CPM 30% 상승 + CTR 15% 하락" 식으로
- [ ] **채널 비교 대시보드** — 한 화면에 모든 채널 ROAS 나란히
- [ ] **액션 추천 엔진** — "CPM이 올랐으니 타겟 범위를 좁히거나 소재를 교체하세요"
- [ ] 소재 **A/B 비교 뷰** — 두 소재를 나란히 놓고 성과 비교

## 라운드 4: 넷째 주 — "상사에게 보고해야 하는데"

### 시나리오
월간 리포트 작성 시간. 대표에게 이번 달 성과를 보고해야 한다.

### 수행 과정
1. 월별 요약 페이지 확인
2. PDF 내보내기
3. 핵심 지표 추출 → PPT에 복사
4. "왜 이런 결과가 나왔는지" 스토리라인 작성

### 발견된 문제점
13. **월별 요약이 너무 단순** — 숫자만 있고 인사이트가 없음
14. **PPT용 차트가 없음** — 대시보드 차트를 캡처해야 함 (해상도 낮음)
15. **전월 대비 비교가 자동이 아님** — 수동으로 날짜 변경해서 비교해야 함
16. **보고서 템플릿이 없음** — 매번 처음부터 작성

### 개선 방안
- [ ] **월간 리포트 자동 생성** — "이번 달 매출 ₩XX, 전월 대비 +XX%, 주요 원인: ..."
- [ ] **PPT 내보내기** 또는 고해상도 차트 이미지 다운로드
- [ ] **기간 비교 모드** — 이번 달 vs 지난 달 나란히
- [ ] **리포트 템플릿** — 대표용/팀용 프리셋

---

## 라운드 5: 다섯째 주 — "퍼널이 어디서 새는 거야?"

### 시나리오
구매 전환율이 낮다. 퍼널 어디서 이탈하는지 찾아야 한다.

### 수행 과정
1. Funnel 탭 확인
2. 채널별 퍼널 비교 (카페24 vs 스마트스토어 vs 쿠팡)
3. 장바구니→구매 전환율 확인
4. 어느 단계에서 이탈이 많은지 분석

### 발견된 문제점
17. **퍼널 데이터가 채널별로 따로** — 전체 퍼널 한눈에 비교가 안 됨
18. **이탈률 시각화가 없음** — 숫자만 있고 "어디서 새는지" 직관적이지 않음
19. **퍼널 단계별 트렌드가 없음** — "장바구니 추가가 줄고 있다" 같은 추이 파악 불가
20. **쿠팡 퍼널 데이터가 자주 비어있음** — 수기 파일 업로드 의존

### 개선 방안
- [ ] **퍼널 시각화** — 깔때기(funnel) 차트로 이탈률 직관적 표시
- [ ] **채널별 퍼널 비교 차트** — 3채널 나란히
- [ ] **퍼널 단계별 일별 트렌드** — 장바구니/구매 각각의 추이
- [ ] 쿠팡 퍼널 자동화 (셀러인사이트 파일 주기적 다운로드 알림)

## 라운드 6: 여섯째 주 — "상품별로 뭐가 잘 팔리지?"

### 시나리오
신제품 런칭 후 성과 확인. 상품별 매출/트렌드가 필요하다.

### 수행 과정
1. Sales 탭 → 상품별 매출 확인
2. 신제품 vs 기존 제품 비교
3. 채널별 상품 성과 차이 확인
4. 재고/원가 대비 수익성 분석

### 발견된 문제점
21. **상품별 트렌드가 없음** — 현재 기간 합계만, 일별/주별 추이 없음
22. **상품 원가가 대부분 미등록** — 66개 중 0개 등록 → 수익성 분석 불가
23. **채널×상품 크로스 분석이 없음** — "덴탈껌이 쿠팡에서 잘 팔리나 카페24에서 잘 팔리나" 못 봄
24. **상품 카테고리별 요약이 없음** — 간식/사료/영양제별 비교

### 개선 방안
- [ ] **상품별 일별 트렌드 차트** (클릭하면 확장)
- [ ] **상품 원가 일괄 등록** 유도 (빈 원가 경고)
- [ ] **채널×상품 히트맵** — 어떤 상품이 어떤 채널에서 잘 팔리는지
- [ ] **카테고리별 대시보드** — 간식/사료/영양제 탭

## 라운드 7: 일곱째 주 — "키워드 뭘 밀어야 하지?"

### 시나리오
네이버 검색광고 키워드 최적화가 필요하다.

### 수행 과정
1. Keywords 탭 확인
2. 캠페인별 성과 비교
3. CPC 높은 키워드 확인
4. 키워드 리서치 실행 (시트에서)

### 발견된 문제점
25. **키워드 탭이 캠페인 뷰만** — 개별 키워드 성과가 없음 (네이버 쇼핑 한계)
26. **키워드 리서치 결과가 대시보드에 안 나옴** — 시트에만 있어서 따로 봐야 함
27. **검색광고 ↔ 매출 연결이 안 됨** — 어떤 키워드가 실제 구매로 이어졌는지 모름
28. **키워드 추천이 없음** — "이 키워드를 추가하면 좋겠다" 같은 제안

### 개선 방안
- [ ] **키워드 리서치 결과 대시보드 통합** — 시트 데이터 연동
- [ ] **키워드 → 매출 기여도** (UTM 기반 추적)
- [ ] **키워드 추천 엔진** — 검색량 대비 경쟁도 낮은 키워드 제안
- [ ] 키워드 **입찰가 최적화 시뮬레이터**

## 라운드 8: 여덟째 주 — "콘텐츠 뭐가 효과 있었지?"

### 시나리오
블로그/인스타 콘텐츠 성과를 확인하고 다음 달 콘텐츠 계획을 세운다.

### 수행 과정
1. Content 탭 확인 → 비어있음
2. 블로그 방문자 수 확인 → 데이터 없음
3. 인스타 성과 → 연결 안 됨
4. 수동으로 각 플랫폼 확인

### 발견된 문제점
29. **Content 탭이 사실상 빈 페이지** — 블로그/SNS 데이터 자동 수집 안 됨
30. **인스타 API 연결 실패** — business_management 권한 미획득
31. **콘텐츠 → 전환 기여도 추적 안 됨** — 어떤 블로그 글이 유입→구매까지 연결됐는지
32. **콘텐츠 캘린더가 없음** — 발행 일정 관리 기능

### 개선 방안
- [ ] **블로그 크론** 실행 (5개 블로그 일별 방문자)
- [ ] **인스타 API 재연결** (business_management 권한)
- [ ] **콘텐츠 성과 테이블** — URL별 조회/유입/전환
- [ ] **콘텐츠 캘린더** 기본 기능

## 라운드 9: 아홉째 주 — "모바일에서 확인해야 하는데"

### 시나리오
외근/미팅 중 모바일로 빠르게 성과를 확인해야 한다.

### 수행 과정
1. 핸드폰으로 대시보드 접속
2. Overview 확인 → 스크롤이 많음
3. 차트가 잘 안 보임
4. 데이터 입력하려고 하는데 불편

### 발견된 문제점
33. **모바일 KPI 카드가 작음** — 핵심 숫자가 한눈에 안 들어옴
34. **차트 터치 인터랙션 불편** — 툴팁이 잘 안 뜸
35. **모바일 데이터 입력이 번거로움** — 숫자 키보드/파일 업로드가 불편
36. **핵심 지표만 빠르게 볼 수 없음** — 전체 페이지를 스크롤해야 함

### 개선 방안
- [ ] **모바일 전용 요약 뷰** — 핵심 KPI 3-4개만 크게
- [ ] **모바일 차트 최적화** — 단순화된 차트
- [ ] **텔레그램 봇으로 빠른 조회** — "/매출" 치면 오늘 매출 반환
- [ ] **PWA 푸시 알림** — 이상치 발생 시 모바일 알림

## 라운드 10: 열째 주 — "팀원이랑 같이 봐야 하는데"

### 시나리오
새 팀원이 들어왔다. 대시보드 사용법을 알려줘야 한다.

### 수행 과정
1. 대시보드 URL 공유
2. 데이터 입력 방법 설명 → 매뉴얼이 없어서 구두 설명
3. 각자 데이터 입력 → 중복 입력 우려
4. 권한 관리 필요

### 발견된 문제점
37. **사용 매뉴얼이 없음** — 새 사람이 혼자 못 씀
38. **입력 이력이 없음** — 누가 언제 뭘 입력했는지 모름
39. **중복 입력 방지가 없음** — 같은 날 두 번 입력하면?
40. **사용자 권한 구분이 없음** — 모든 사람이 모든 걸 수정 가능

### 개선 방안
- [ ] **인앱 매뉴얼/온보딩** — 첫 접속 시 가이드 투어
- [ ] **입력 로그** — 누가 언제 뭘 입력했는지 히스토리
- [ ] **중복 입력 경고** — "이 날짜 데이터가 이미 있습니다. 덮어쓸까요?"
- [ ] 기본적인 **역할 구분** (관리자/뷰어)

## 라운드 11: 열한째 주 — "자동화 좀 더 하고 싶은데"

### 시나리오
반복 작업이 지겨워졌다. 더 자동화하고 싶다.

### 수행 과정
1. 매일 같은 데이터 입력 루틴
2. "이거 왜 자동이 안 되지?" 의문
3. GFA, 스마트스토어는 API가 없어서 수기 불가피
4. 그래도 줄일 수 있는 게 있지 않나?

### 발견된 문제점
41. **매일 5-10분 소요** — 파일 다운로드 → 업로드 → 수기 입력 루틴
42. **카페24 데이터는 자동화 가능한데 안 되어있음** — 회원가입 수, 주문 수 등
